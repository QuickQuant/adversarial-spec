# Gemini Review Instructions: adversarial-spec

## Section 1: What You're Looking At

**adversarial-spec** is a Claude Code skill (CLI tool) that refines product specifications through multi-model debate and adversarial stress-testing. It sends specs to multiple LLMs in parallel (GPT, Gemini, Claude, Grok, Mistral), collects critiques, and iterates until consensus. A gauntlet mode runs specs through named adversary personas (security paranoid, burned oncall, UX skeptic, etc.) in a 7-phase attack-evaluate-rebuttal pipeline.

- **Runtime:** Python 3.14+
- **Dependencies:** uv, litellm, pydantic
- **Components:** 9 (debate engine, gauntlet, models, providers, prompts, adversaries, pre-gauntlet, MCP tasks, execution planner)
- **Total source:** ~27,755 lines across 40+ Python files
- **Tests:** 7 test files, ~4,582 lines

## Section 4: Key Architecture Concepts

**Primary data flow:** User invokes CLI → debate.py parses args → routes to action (critique/gauntlet/session) → calls LLMs in parallel via ThreadPoolExecutor → collects responses → saves session state → reports results.

**Design patterns:**
- **Single-invocation CLI** — no daemon, each command is one round or one gauntlet run. Sessions enable multi-round iteration.
- **Thread-per-model parallelism** — ThreadPoolExecutor dispatches to all models simultaneously.
- **4 LLM calling routes** — LiteLLM API (7+ providers), Codex CLI subprocess, Gemini CLI subprocess, Claude CLI subprocess.
- **Checkpoint persistence** — JSON files after each gauntlet phase and debate round for crash recovery.
- **Layered dependencies** — CLI → Orchestration → LLM → Config. No circular imports.
- **Isolated subsystems** — pre-gauntlet has zero imports from debate/gauntlet modules.

**Critical constraint:** CLI model names are passed to subprocess commands. While list-based `subprocess.run` prevents shell injection, model names aren't character-validated.

## Section 6: How to Work With This Codebase

```bash
# Test
uv run pytest

# Lint
uvx ruff check --fix --unsafe-fixes

# Run debate
uv run debate critique --spec path/to/spec.md --models gpt-5.3,claude-opus-4-6

# Run gauntlet
uv run debate gauntlet --spec path/to/spec.md
```

**File conventions:**
- All scripts in `skills/adversarial-spec/scripts/`
- Phase definitions in `skills/adversarial-spec/phases/`
- Tests alongside source in `scripts/tests/`
- No circular imports between components

**Critical constraints:**
- Never log API keys or config objects containing secrets
- Always use `uv run` for Python commands
- Changes to `skills/adversarial-spec/` need manual copy to `~/.claude/skills/`
