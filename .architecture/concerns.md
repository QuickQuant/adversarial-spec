## Top Concerns

### CON-001: tasks.json concurrent write hazard

- **Priority:** now
- **Severity:** warning
- **Component:** mcp-tasks
- **Confidence:** medium
- **Why it matters:** MCP server and TaskManager both read/write `.claude/tasks.json` without locking. In multi-agent workflows (Claude + Codex working in parallel), concurrent writes cause last-write-wins data loss. This is the only medium-severity hazard in the codebase.
- **Recommended action:** Add FileLock wrapper to `load_tasks()`/`save_tasks()` in both `mcp_tasks/server.py` and `task_manager.py`. filelock is already a dependency (used by gauntlet persistence).
- **Files:**
  - `mcp_tasks/server.py`
  - `skills/adversarial-spec/scripts/task_manager.py`
- **Source refs:** HAZ-003, FIND-003

### CON-003: Unify model dispatch abstractions

- **Priority:** next
- **Severity:** warning
- **Component:** models, gauntlet
- **Confidence:** high
- **Why it matters:** Three competing model dispatch paths (`model_dispatch.call_model()`, `models.call_models_parallel()`, and phase_2 inline dispatch) make it impossible to apply improvements uniformly. Adding providers or changing retry logic requires multi-file updates.
- **Recommended action:** Unify on single `call_model()` interface in `model_dispatch.py`, route all gauntlet phases through it.
- **Files:**
  - `skills/adversarial-spec/scripts/gauntlet/model_dispatch.py`
  - `skills/adversarial-spec/scripts/gauntlet/phase_2_synthesis.py`
  - `skills/adversarial-spec/scripts/models.py`
- **Source refs:** call-model-abstraction-inconsistency

### CON-004: Silent error swallowing in gauntlet phases

- **Priority:** next
- **Severity:** warning
- **Component:** gauntlet
- **Confidence:** high
- **Why it matters:** Generic `except` handlers in 3 gauntlet phases catch all errors and return safe defaults. Downstream phases proceed with degraded data, making failures invisible until the final verdict is unexpectedly weak.
- **Recommended action:** Catch specific exceptions (API errors, timeouts, JSON parse errors). Let unexpected errors propagate. Log recoverable errors at WARNING level.
- **Files:**
  - `skills/adversarial-spec/scripts/gauntlet/phase_1_attacks.py`
  - `skills/adversarial-spec/scripts/gauntlet/phase_4_evaluation.py`
  - `skills/adversarial-spec/scripts/gauntlet/phase_5_rebuttals.py`
- **Source refs:** try-except-with-generic-fallback

### CON-005: Fix test infrastructure

- **Priority:** next
- **Severity:** warning
- **Component:** gauntlet
- **Confidence:** high
- **Why it matters:** 7 test files repeat identical `sys.path.insert` preamble for imports (fragile, non-idiomatic), and there are no end-to-end gauntlet pipeline tests — phase-to-phase integration may break silently.
- **Recommended action:** (1) Configure `pyproject.toml` `[tool.pytest.ini_options] pythonpath` to eliminate sys.path hacks. (2) Add integration tests with mocked models but real data structures flowing through all 7 phases.
- **Files:**
  - `scripts/tests/test_gauntlet_phase_1_attacks.py`
  - `scripts/tests/test_gauntlet_core_types.py`
  - `scripts/tests/test_gauntlet_persistence.py`
  - `scripts/tests/test_providers.py`
  - `scripts/tests/test_model_calls.py`
  - `scripts/tests/test_cli.py`
  - `scripts/tests/test_gauntlet_orchestrator.py`
- **Source refs:** sys-path-insertion-in-test-files, missing-integration-tests

### CON-006: Pydantic dependency not declared

- **Priority:** next

- **Component:** pre-gauntlet
- **Confidence:** high
- **Why it matters:** `pre_gauntlet/models.py` uses Pydantic BaseModel but pydantic is not in `pyproject.toml`. Works only because pydantic happens to be installed in the environment.

### CON-007: Centralize gauntlet prompts

- **Priority:** later
- **Severity:** warning
- **Component:** gauntlet
- **Confidence:** medium
- **Why it matters:** 4 gauntlet phase modules hardcode system/user prompts inside phase functions. Scattered prompts are hard to audit, version, or adjust without touching phase logic.
- **Recommended action:** Move prompts to a dedicated `gauntlet/prompts.py` module. Keep phase functions focused on orchestration.
- **Files:**
  - `skills/adversarial-spec/scripts/gauntlet/phase_1_attacks.py`
  - `skills/adversarial-spec/scripts/gauntlet/phase_4_evaluation.py`
  - `skills/adversarial-spec/scripts/gauntlet/phase_5_rebuttals.py`
  - `skills/adversarial-spec/scripts/gauntlet/phase_6_adjudication.py`
- **Source refs:** prompt-template-duplication-across-phases

### CON-008: Gauntlet CLI flag divergence

- **Priority:** later
- **Severity:** warning
- **Component:** gauntlet
- **Confidence:** high
- **Why it matters:** `debate.py` uses `--codex-reasoning`/`--gauntlet-resume` while `gauntlet/cli.py` uses `--attack-codex-reasoning`/`--resume`. Users hitting the wrong CLI get confusing errors.
- **Recommended action:** Add flag aliases so both spellings work in both CLIs, or document the mapping in `--help` output.
- **Files:**
  - `skills/adversarial-spec/scripts/debate.py`
  - `skills/adversarial-spec/scripts/gauntlet/cli.py`
- **Source refs:** FIND-005
