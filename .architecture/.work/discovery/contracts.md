## DATA_MODEL_SURFACES

### Checkpoint Storage (.adversarial-spec-gauntlet/)
- Envelope: {_meta: {schema_version, spec_hash, config_hash, phase, created_at, data_hash}, data: {...}}
- Protected by FileLock
- Atomic write via temp file + rename

### Adversary Stats (~/.adversarial-spec/adversary_stats.json)
- Per-adversary: concerns_raised, accepted, dismissed, signal_score, avg_concern_length
- Updated after each gauntlet run

### Medals Database (~/.adversarial-spec/medals/)
- medal-{concern_id}-{timestamp}.json
- Medal.to_dict() schema

### Resolved Concerns (~/.adversarial-spec/resolved_concerns.json)
- dict[concern_id -> {verdict, reasoning, ...}]
- Caches decisions to skip re-evaluation

### Session Checkpoints (.adversarial-spec-checkpoints/)
- {session_id}-round-{N}.md (spec)
- {session_id}-round-{N}-critiques.json (responses)

### Task Store (.claude/tasks.json)
- Shared with MCP Tasks server
- {tasks: list[Task.to_dict()], next_id: int}
